浅谈复式记账与 Beancount
2021-02-28 15:59 folder [Accounting]   label accounting   beancount   python   schedule 7 min 31 s.

复式记账法是世界通行的会计方法， Beancount 是采用复式记账法的工具，由 Python 编写，定位个人记账。 起因是创作者不满于现存的记账工具，于是独自开发了 Beancount，并开源贡献给所有人使用， 非常有 Geek 风范。

本文的目的，是简要了解复式记账法的原理，以及 Beancount 是如何应用这些原理的。

复式记账

记账

交易，是商业的本质。 无论是以物易物，还是使用通用货币，或者 App 中数字的增减，这一点从没有发生变化。

记账，就是记录交易。 交易必定涉及两方，每方都要准确记录交易标的，交易金额，明确自身的财务情况。

复式记账法是当前主流的记账方式，而且不限交易主体。 无论是个人还是公司，都可以采用复式记账法。 区别只在于两种主体的交易项目不同，金额数目不同，交易的本质并没有变化。

账户

凡记账，必须要设立账户。

复式记账法的特色在于 每笔交易都以相等的金额，同时登记在两个账户中，清晰反映了资金的流转过程。

如何设立账户可以同时登记？会计恒等式已经给出了方案。

资产 = 负债 + 所有者权益

恒等式的三项，将所有账户分为三大类，简单理解即

  • 资产，表面上你所拥有的
  • 负债，不属于你的，迟早要归还的
  • 所有者权益，真正属于个人的

在三类账户下，细分设立更多的子账户，用以记录交易。 同时恒等意味着，公式在任意时刻， = 必须成立，不然就是记账出现问题。

下面以一个公司成立之初的几笔交易，来了解复式记账的过程。

公司成立，创始人注入资本。

创始人的注入的资本，是真正属于公司的，属于所有者权益，细分账户实缴资本。

注入的资金为公司所有，表面在资产分类上，细分账户现金。

向银行借款。

公司向银行借款，属于负债，细分应付票据。

同时借来的现金扩大了公司的资产。

购买存货。

假如公司本身不进行生产，只进行倒卖。

公司花钱购买一批存货，准备日后出售。 此时现金减少，存货增加。

存货属于资产分类，资金只在资产账户下面流动。

可以观察到，等式永远是成立的。 同时资金的来源，去向，账户的余额都一目了然。

借？贷？

会计中有两个动词，借和贷，两者的区别困扰了很多人。

解除这个困惑，要从会计的流程开始。

究其根本，会计的工作，就是将所有单项交易，最终整合为报表，呈现公司的财务整体面貌。

其中的分类账，使用一种简化的 T 形账户来登记，因为形状像 T 而得名。

每个 T 形账户分为左右两边，分别表示资金的增加和减少。

优点在于，每次登记时，只记录相关的账户，不用把整个恒等式搬出来。

同时每次记录的恒等关系变成 T 形左边 = T 形右边 。

如果账户的左边代表增加，右边表示减少，上面的记录方式是错误的。

所以不同类型的账户，左右的含义是不同的。恒等式 = 两边的账户是相反的。

借(debit) 和贷(credit) 的含义非常纯粹，表示账户的左边和右边。 正因为有时“借”，金额增加了，有时金额减少了，才造成了不少人的疑惑， 其实是账户类型的不同影响的。

收入账户与费用账户

除此之外，还有两个账户比较常用，收入账户 和 费用账户。

这两个是留存权益的子账户，单独设立出来，方便计算利润。

“收入” 和 “费用” 都是正数，但是含义不同。 收入增加，提高了留存收益；费用增加，减少了留存收益。

假如公司转手销售存货，

同时得到收入

利润 = 收入 - 费用

留存收益提升了 30,000 。

Beancount

Beancount 是 Python 开发的工具，使用复式记账法，用于个人记账。

$ pip install beancount fava

推荐 byvoid 和 wzyboy 的文章，将 Beancount 的用法讲解的非常透彻， 这里不再赘述。

下面简要谈谈 Beancount 和复式记账的对应关系。

账户

Beancount 内建有 5 个账户，和上面的讲解是对应的，

  • Assets，资产
  • Liabilities，负债
  • Equity，所有者权益
  • Income，收入
  • Expenses，费用

恒等式

Beancount 内部同样遵循会计恒等式，不过进行了移项处理。

资产 + 负债 + 所有者权益 = 0

这样带来的结果是，负债和所有者权益的相关账户都发生了颠倒， 这一点需要用户来适应。

借贷关系

Beancount 中使用 + 表示借， - 表示贷，不关心账户的正负值， 这一点需要结合上一点来适应。

End

相信有了复式记账的理解以及 Beancount 中的些许差异， 可以帮助读者在记账过程中减少一些困惑。
