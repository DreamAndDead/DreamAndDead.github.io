枚举排列 组合是常见的情况

单独讲解

* 集合

排列与组合都涉及到一组元素，称这组元素为集合（set）。
排列是集合所有元素的一种顺序；
组合是集合的子集。

存在一种集合的推广，称为多重集合（multiset）[fn:1]。
在一个集合（set）中，相同的元素只能出现一次；
在多重集（multiset）之中，同一个元素可以出现多次。

集合类别不同，枚举的方法也存在些许差异。

* 排列

排列是集合中所有元素对应的一个有序序列。

有两种常用方式来枚举排列，迭代法和递归法。

** 迭代法

迭代法和字典序相关。

字典序，义如其名，如字典中单词一样的排序规则，
- 两个单词（序列）从左到右，依次比较字符（元素），直到比出大小
- 空字符（元素）最大

考虑集合 ={1, 2, 3, 4}= ，存在 16 个排列序列。

将所有序列按照字典序从小到大排序，
其中相邻的两个序列有明显的关系，
迭代法枚举排列利用的就是这一点。

第一个最小的序列是已知的，将元素排序即可， =1 2 3 4= 。
然后不断枚举字典序中相邻的下一个序列。

如何找出下一个序列？

根据字典序的定义，左边的元素影响大于右边的元素。
在寻找相邻的下一个序列时，从尾部开始变动，尽量不变动头部的元素。

以序列 =1 2 3 4= 为例，
- 最后一位为 4，只有一个数，没有变动的空间，跳过
- 最后两位为 =3 4= ，因为 =3 < 4= ，则可以交换两者，得到下一个字典序 =1 2 4 3=

以序列 =1 2 4 3= 为例，
- 最后一位，跳过
- 最后两位， =4 3= ， =4 > 3= ，已经没有可调整的空间，使字典序变大，只能继续向前
- 最后三位， =2 4 3= ，此时 =2 < 4= ，但是不能直接交换 2 4，交换 2 3 是更好的选择，
  因为在最后两位中， 3 是比 2 大的最小的数，而不是 4。
  高位确定之后，得到 =1 3 4 2= ，
  这个序列虽然比 =1 2 4 3= 大，但不是最小的，
  此时需要将低两位排序，得到比 =1 2 4 3= 大的最小的字典序 =1 3 2 4= ，即与之相邻。

*如何寻找替换高位的最小元素？*

在不得已替换高位时，可以发现，高位后的低位问题逆序排列的情况。
如果不是逆序，说明存在不需要替换高位的更大的字典序。

=1 4 3 2= 需要替换 高位 1，但是 =1 4 2 3= 就不需要替换高位。

所以只需要倒序依次查找，直到找到比高位大的最小元素即可。

*替换高位后，所有低位必须要进行 sort 过程？*

仔细考虑其实不需要，因为低位已经是逆序，交换高位后，逆序的性质并没有改变。
此时只需要 reverse 低位，就可以得到低位的排序。

#+begin_src c++
bool next_perm(int perm[], int n) {
  for (int i = n-1; i > 0; i--) {
    int j = i - 1;

    if (perm[i] > perm[j]) {
      int k;
      for (k = n-1; k >= i; k--) {
	if (perm[k] > perm[j])
	  break;
      }

      swap(perm[j], perm[k]);

      reverse(perm + i, perm + n);

      return true;
    }
  }

  return false;
}
#+end_src
